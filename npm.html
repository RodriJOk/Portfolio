Introduccion a Npm - Guia basica

En este ultimo tiempo, hemos visto como Node a tomado mucha popularidad en el mundo del desarrollo en el lenguaje Javascript. Lo interesante es que no podemos hablar de Node.Js y pasar por alto NPM. Es por eso que en este articulo hablaremos de el a profundidad. Espero que te sea de mucha utilidad. üòä

NPM - ¬øQue es?

NPM es un gestor de paquetes üì¶, el mas popular que tiene Javascript, donde encontaras una gran cantidad de recursos para poder implentar en tus proyectos. Tambien vas a poder crear tus propios paquetes y compartilos con toda la comunidad. En su sitio podras ver a detalle mas informacion sobre los paquetes que podras implementar en tus proyectos. Desde librerias open source hasta grandes proyectos. NPM (por sus siglas en ingles Node Package Mananger) se encarga de administar todos los paquetes y, algo importante que debemos tener en cuenta, es que para trabajar con NPM, debemos tener instalada la version de Node.

Pero, ¬øComo hacemos eso?. Tranquilo/a üòâ. Es muy simple, aqui veras una guia de pasos para llevar a cabo.

En Windows, sigue estos pasos:

1_ Descarga e instalacion de Node: Para poder empezar a trabajar con NPM, necesuitamos tener instalado Node.js en nuestro computador. Para ello vamos a su sitio oficial y descargamos Node.

Una vez descargado Node, pasaremos a la etapa de instalacion. (El famoso Next -> Next üòÖ). Una vez terminada la instalacion, podemos chequear si tenemos Node en nuestras computadoras si habriamos la consola (Simbolo del sistema) y podemos ejecutar el comando node -v. Si como resultado, vemos el numero de la version de Node.js, lo habras instalado correctamente üôåüèª.

Tambien podemos comprobar que tenemos instalado NPM, si ejecutamos el comando npm -v y podemos ver la version del mismo

Si queremos asegurarnos de que tenemos la ultima versiom podemos ejecutar el comando npm install -g npm@lastest

Inicia un proyecto con npm

Es de mucha importancia de que cuando vayas a crear un proyecto, lo inicialices con npm. Esto nos va a permitir generar una cierta configuracion con ciertos datos importantes para nuestro entorno, asi como tambien compartirlo con otros desarrolladores y puedan tener una nocion de lo que esta ocurriendo, para eso nos vamos a mover a nuestra terminal, en donde vamos a inicilaizar nuestro primer proyecto con NPM. Como muestra de ejemplo, voy a crear una carpeta con el nombre NPMjs y voy a posicionarme dentro de el con el comando cd "nombrearchivo".

Una vez creada nuesta carpeta del proyecto, ahora si estamos listos para poder inicializar nuestro proyecto con NPM. Pero ¬øcomo? ¬°Es muy sencillo! Solo debemos ejecutar el comando npm init. Al ejecutar este comando, nos va a permitira nosotros, crear el archivo package.json, con el cual vamos a tener una configuracion establecida, una descripcion del proyecto y cierto valores que necesita.

Al ejecutar el comando tambien nos aparecera una peque√±a introduccion, con algunas explicaciones. Y tambein nos permitira ir completando la informacion con los valores que desemos (con algunas sugerencias).

Los valores que vamos a tener que configurar son los siguientes

  1. Nombre del paquete (package.name): Aqui debemos colocar el nombre de nuestro proyecto

  2. Version: Numero de version de nuestro proyecto. La sugerencia es la version inicial (1.0.0)

  3. Descripcion (Description): En este campo podemos colocar una breve descripcion de nuestro proyecto

  4. Punto de entrada (entry point): Este paso es importante, por que aqui vamos a definir nuestro punto de entrada de nuestro proyecto.

  5. Test command: Aqui nosotros podremos configurar comandos de utilidad para nuestro proyecto.

  6. Repositorio de Git (git): Aqui podemos establecer el nombre de nuestro repositorio de Git

  7. Palabras claves (keywords): Aqui podremos establecer palabras claves que tengan relacion o describan a nuestro proyecto.

  8. Autor (Author): Aqui podremos establecer quien es la persona que esta construyecndo este proyecto inicialmente.

  9. Licencia (license): Aqui podremos configurar la licencia que tendra nuestro proyecto. En este aspecto podremos elegir entre MIT o ISC.

Luego de establecer los valores, tendremos un resumen con todas las configuraciones que hemos establecido. Si luego de darle un vistazo esta todo correcto, podemos terminar la configuracion de nuestro proyecto.

Luego de terminado todos estos pasos, vamos a poder ver en nuestro proyecto un archivo llamado package.json
